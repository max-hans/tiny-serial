# bun-serial

Cross-runtime serial port library (Node.js + Bun). "Thin Native, Fat TypeScript": Rust handles only OS syscalls; TypeScript owns streams, parsers, and the mock API.

## Architecture

- **`src/lib.rs`** — Rust. Opens ports, background reader thread via `Arc<ThreadsafeFunction<Buffer>>` (napi v3), async writes via `AsyncTask`. Uses `try_clone()` so reader and writer have independent handles.
- **`lib/`** — TypeScript source. Compiled to `dist/` by `tsc -p tsconfig.build.json`.
- **`index.js` / `index.d.ts`** — napi-rs owned. Regenerated on every build. Do not edit.
- **`serial.js` / `serial.d.ts`** — Hand-authored public entry point → `dist/index.js`.

## Commands

```bash
bun run build:debug          # Rust (debug) + TypeScript
bun run test                 # AVA (Node) — no hardware needed
bun run test:bun             # bun test — no hardware needed
./scripts/test-hardware.sh   # native tests with socat virtual ports
```

## Rules

- Never edit `index.js` or `index.d.ts` — they are regenerated by `napi build`.
- `lib/SerialPort.ts` uses dynamic `import()` (not `require`) for the native binding so it works in both ESM (tests) and CJS (compiled output).
- `MockSerialPort` has zero native imports — keep it that way so it runs anywhere.
- All parser and mock tests must pass without a compiled `.node` binary.
- Package manager is **bun**. Use `bun install`, `bun run`, `bun test`.
- Rust needs `~/.cargo/bin` in PATH. Build scripts prefix it automatically.

## Adding a Parser

1. Create `lib/parsers/YourParser.ts` extending `Transform` from `node:stream`.
2. Export it from `lib/index.ts`.
3. Add tests to `__test__/parsers.spec.ts` and `__test__/bun-compat.test.ts`.
